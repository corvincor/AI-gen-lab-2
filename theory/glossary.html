База даних (Database)
Організована сукупність даних, призначена для зберігання, пошуку, оновлення та обробки інформації.
Приклад: база даних учнів школи або каталог товарів інтернет-магазину.
Система керування базами даних (СУБД)
Програмне забезпечення, яке дозволяє створювати, змінювати, зберігати та адмініструвати бази даних.
Приклади: MySQL, PostgreSQL, MongoDB.
SQL
Structured Query Language — стандартна мова для роботи з реляційними базами даних.
SELECT * FROM Students;
NoSQL
Клас нереляційних баз даних, що дозволяють працювати з великими обсягами структурованих і неструктурованих даних.
MongoDB, Cassandra, Neo4j.
Реляційна база даних
База даних, у якій інформація зберігається у вигляді таблиць, що пов'язані між собою.
Нереляційна база даних
База даних, яка використовує інші моделі зберігання інформації: документи, графи, ключ-значення або колонки.
Документ
Основна одиниця зберігання інформації у документно-орієнтованих базах даних. 
{
  "name": "Марія",
  "age": 16
}
JSON
JavaScript Object Notation — текстовий формат представлення даних, який широко використовується в NoSQL-системах.
Колекція (Collection)
Набір документів у документно-орієнтованій базі даних.
Колекція «Учні» може містити документи про кожного учня.
Key-Value база даних
NoSQL-модель, у якій дані зберігаються як пари «ключ → значення». 
theme → dark
language → uk
Документно-орієнтована база даних
База даних, що зберігає інформацію у вигляді документів JSON.
MongoDB є найпоширенішою документною NoSQL БД.
Графова база даних
База даних, що представляє інформацію у вигляді вузлів та зв'язків.
Соціальні мережі використовують графові структури для зберігання друзів.
Колонкова база даних
База даних, у якій інформація зберігається за стовпцями, а не за рядками.
Використовується для аналітики та роботи з великими даними.
Вузол (Node)
Основний елемент графової бази даних, який представляє об'єкт.
Учень, книга або місто можуть бути вузлами графа.
Зв'язок (Relationship)
З'єднання між вузлами графової бази даних.
Марія → друг → Андрій
Big Data
Надзвичайно великі обсяги даних, які складно обробляти традиційними способами.
Масштабованість (Scalability)
Здатність системи ефективно працювати при збільшенні кількості користувачів та даних.
Горизонтальне масштабування
Додавання нових серверів для збільшення продуктивності системи.
Вертикальне масштабування
Підвищення потужності існуючого сервера шляхом модернізації його обладнання.
Запит (Query)
Команда для отримання, додавання або зміни інформації в базі даних.
MongoDB
Популярна документно-орієнтована NoSQL база даних, що використовує JSON-подібний формат BSON.
Cassandra
Розподілена колонкова NoSQL база даних, створена для роботи з великими обсягами інформації.
Neo4j
Відома графова база даних для роботи зі складними зв'язками.
Firebase
Хмарна платформа для розробки мобільних і вебзастосунків, що містить документні бази даних Firestore та Realtime Database.
