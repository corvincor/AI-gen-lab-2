Закріпити знання про NoSQL-бази даних, навчитися аналізувати структуру даних, працювати з JSON-документами та обирати відповідний тип бази даних для різних завдань.

Середній рівень

Завдання 3. Аналіз ситуацій

Визначте, який тип NoSQL-бази даних найкраще підходить для кожної ситуації.

  1. Соціальна мережа повинна зберігати друзів та зв'язки між користувачами.
  2. Інтернет-магазин має каталог із тисячами товарів та різними характеристиками.
  3. Сервіс аналітики обробляє мільярди записів щодня.
  4. Вебсайт зберігає налаштування користувачів для швидкого доступу.
Середній рівень

Завдання 4. Робота з JSON

Створіть JSON-документ для учня школи.

Документ повинен містити:

  • ім'я;
  • вік;
  • клас;
  • список предметів;
  • місто проживання.

Приклад структури:

{
  "name": "",
  "age": 0,
  "class": "",
  "subjects": [],
  "city": ""
}

Завдання 5. CRUD-операції

Уявіть, що у вас є колекція students. Опишіть словами або псевдокодом:

  3. Як змінити вік учня.
  4. Як видалити запис про учня.

Завдання 7. Проєктування бази даних для онлайн-бібліотеки

Уявіть, що необхідно створити систему для онлайн-бібліотеки.

Необхідно:

  1. Визначити тип NoSQL-бази даних.
  2. Пояснити свій вибір.
  3. Створити приклад документа JSON для книги.
  4. Передбачити зберігання:
    • назви книги;
    • автора;
    • жанру;
    • року видання;
    • списку ключових слів.

Завдання 10. Розробка концепції інформаційної системи

Запропонуйте власну інформаційну систему, яка могла б використовувати NoSQL.

Наприклад:

  • соціальна мережа;
  • музичний сервіс;
  • система доставки їжі;
  • онлайн-магазин;
  • освітня платформа.

Опишіть:

  1. Призначення системи.
  2. Які дані будуть зберігатися.
  3. Який тип NoSQL буде використано.
  4. Чому саме цей тип є найкращим.
